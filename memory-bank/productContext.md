# Product Context - 产品体验规范

## V1.1 全屏无滚动（Viewport-Fit）视觉宪法

游戏界面必须完整容纳在当前可视视口内。无论 PC、平板或手机，游戏进行页都绝对禁止出现纵向滚动条；用户无需滚动即可看到状态栏、完整棋盘和主要操作控件。

### 布局硬约束

* 页面根节点与游戏主容器使用视口高度约束（如 `h-screen`、`h-[100dvh]`）并设置 `overflow-hidden`。
* 主容器采用纵向弹性布局；状态栏与控制区占用确定空间，棋盘区域使用 `flex-1 min-h-0` 消化剩余高度。
* 棋盘宽高同时受视口限制，必须按可用空间等比缩放；可使用 `max-h-[calc(100vh-200px)]` 或等价的动态视口计算，不能依赖会把页面撑高的固定尺寸。
* 卡牌尺寸必须由棋盘可用宽高共同决定，间距、内边距和圆角随断点收缩，不能通过页面滚动容纳完整棋盘。
* 模态框和浮层必须限制在视口内；内容超限时优先压缩间距与字号层级，不得使游戏主页面产生纵向滚动。
* 移动端优先使用动态视口单位 `dvh`，避免浏览器地址栏变化造成溢出；`vh` 仅作为兼容回退。

### 汉字可读性硬约束

* 卡牌内汉字是首要视觉元素，字号必须全面放大：小屏至少 `text-3xl`，大屏使用 `text-5xl` 或 `text-6xl`。
* 字号放大后仍须保持单字居中、不换行、不裁切；必要时应先缩小卡牌间距和外围留白，而不是牺牲汉字可读性。

### 验收标准

* PC 与移动端常见视口中，`document.documentElement.scrollHeight` 不得大于可视视口高度。
* 16 宫格及所有受支持棋盘必须完整可见，底部操作区不可被折叠或挤出屏幕。
* 所有卡牌汉字在默认缩放比例下清晰可辨，并满足上述响应式字号下限。
