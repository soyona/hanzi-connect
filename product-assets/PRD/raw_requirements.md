# 汉字偏旁部首连连看游戏 - 原始需求单 (Raw Requirements)

## 1. 核心教育目的
- 目标受众：小学中低年级学生（一至三年级）。
- 教学痛点：小学生单纯记汉字很枯燥，容易混淆形近字。
- 核心手段：通过“偏旁”加“部首/剩余部件”的连连看消除游戏，强化汉字是“模块化组合”的认知，在娱乐中提高汉字储备量。

## 2. 核心玩法设计
- 游戏开始：界面展示一个卡牌矩阵（关卡可选 4x4 或 6x6 规模）。
- 卡牌类型：卡牌分为两类，一类是“左偏旁/上部首”（如：氵、亻、辶、𥘵），另一类是“右部件/下部件”（如：每、亍、曼、工）。
- 消除规则：
  - 玩家点击第一张卡牌（变色高亮），再点击第二张卡牌。
  - 如果两张卡牌能拼出一个“合法的常用汉字”（如 “氵”+“每”=“海”，“氵”+“工”=“江”），则两张卡牌同时消除，玩家得分。
  - 如果不能拼出汉字，或者两张卡牌是同类型的（比如两个都是左偏旁），则判定失败，扣除时间或生命值，卡牌重置。
- 胜利条件：矩阵中所有卡牌被完全消除，进入下一关。

## 3. 核心限制与黑名单（非常重要）
- 🚫 严禁生僻字：拼字判定必须严格基于“教育部小学常用字库”。如果两个部首能拼出一个极其生僻的古汉语汉字，但在小学阶段根本不学，应当判定为无效，避免误导孩子。
- 🚫 拒绝复杂状态：为了保证独立开发效率，第一版（V1.0）不需要做任何全球排行榜、不需要做用户注册、不接入任何真实的后端数据库，所有的关卡进度和最高分完全存在浏览器的 LocalStorage 里。
- 🚫 拒绝花哨特效：纯前端 Canvas 或 DOM 渲染，音效和动画最简化，核心在于汉字匹配算法的准确性。

## 4. 技术栈锁死
- 架构：Vite + React + TypeScript + Tailwind CSS。
- 逻辑：所有汉字组合的合法性校验，全部在前端通过一个静态的代码映射表（字典 JSON）实现。
